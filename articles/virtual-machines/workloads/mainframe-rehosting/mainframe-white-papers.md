# <a name="azure-white-papers-about-mainframe-topics"></a>White papers do Azure sobre tópicos de mainframe

Obtenha recursos sobre migração de mainframe, rehospedageação e movimentação de sistemas baseados em Z da IBM para o Microsoft Azure.

### <a name="demystifying-mainframe-to-azure-migration"></a>[Desmistificando mainframe para migração do Azure](https://azure.microsoft.com/resources/demystifying-mainframe-to-azure-migration/)

O Azure oferece uma alternativa de mainframe capaz de oferecer funcionalidades e recursos equivalentes. Este artigo, de Larry Mead, da equipe AzureCAT, abrange os principais componentes de mainframe z/OS da IBM e os equivalentes do Azure. Ele também fornece um roteiro para iniciar a conversa com os tomadores de decisão de TI que assinam filosofias de mainframe desatualizadas.

### <a name="move-mainframe-compute-and-storage-to-azure"></a>[Mova computação e armazenamento de mainframe para o Azure](https://azure.microsoft.com/resources/move-mainframe-compute-and-storage-to-azure/)

Para executar cargas de trabalho de mainframe no Microsoft Azure, você precisa saber como os recursos do seu mainframe se comparam ao Azure. Baseado em um mainframe IBM z14, este guia de Larry Mead da AzureCAT explora como obter resultados comparáveis no Azure. Os recursos de computação e armazenamento maciçamente escaláveis no Azure fornecem benefícios imediatos para as organizações que executam cargas de trabalho de mainframe.

### <a name="microsoft-azure-government-cloud-for-mainframe-applications"></a>[Nuvem do governo do Microsoft Azure para aplicativos de mainframe](https://azure.microsoft.com/resources/microsoft-azure-government-cloud-for-mainframe-applications/)

Planejar uma migração de aplicativos é o momento ideal para agregar valor e agilidade até mesmo a cargas de trabalho de mainframe bem estabelecidas. Neste guia rápido, Larry Mead, do AzureCAT, mostra como as agências governamentais dos Estados Unidos e seus parceiros podem usar o Governo Azure para aplicativos de mainframe — e a migração pode não ser tão difícil quanto você pensa. O Governo Do Azul oferece as vantagens de um mainframe em um ambiente mais econômico e ágil. Além disso, o governo azure ganhou uma Autoridade Provisória para Operar (P-ATO) para o FedRAMP De Alto Impacto.

### <a name="deploy-ibm-db2-purescale-on-azure"></a>[Implantar o IBM DB2 pureScale no Azure](https://azure.microsoft.com/resources/deploy-ibm-db2-purescale-on-azure/)

Aprenda com nossa experiência com uma empresa que recentemente reorganizou seu ambiente IBM DB2 no Azure. Este guia foi escrito pelos membros da equipe AzureCAT e DMJ que estavam lá: Larry Mead, Benjamin Guinebertière, Alessandro Vozza e Jonathon Frost. Eles descrevem os passos que tomaram durante esta migração. Suas descobertas foram revisadas por membros da equipe pureScale do IBM Db2. Os scripts de instalação, disponíveis no GitHub, são baseados na arquitetura que a equipe usou para uma carga de trabalho OLTP de médio porte típica.

### <a name="install-tmaxsoft-openframe-on-azure"></a>[Instale o TmaxSoft OpenFrame no Azure](https://azure.microsoft.com/resources/install-tmaxsoft-openframe-on-azure/)

Modernize sua infra-estrutura em escala de nuvem. O TmaxSoft OpenFrame facilita a elevação dos ativos existentes do mainframe e os transfere para o Azure. Neste artigo, Steve Read (AzureCAT) e Manoj Aerroju (TmaxSoft) explicam como configurar um ambiente OpenFrame adequado para o desenvolvimento, demonstrações, testes e cargas de trabalho de produção.

### <a name="ibm-mainframe-to-microsoft-azure-reference-architecture"></a>[Mainframe da IBM para arquitetura de referência do Microsoft Azure](https://www.astadia.com/whitepaper/ibm-mainframe-to-microsoft-azure)

Este white paper reflete os mais de 25 anos de experiência em modernização de plataformas de mainframe da Astadia. Eles explicam os benefícios e desafios dos esforços de modernização. Este guia fornece uma visão geral do mainframe da IBM e de uma arquitetura de referência do mainframe para azure da IBM. Também fornece um olhar para a metodologia de sucesso da Astadia.

### <a name="deploying-mainframe-applications-to-microsoft-azure"></a>[Implantando aplicativos de mainframe no Microsoft Azure](https://www.microfocus.com/media/white-paper/deploying_mainframe_applications_to_microsoft_azure_wp.pdf)

As soluções da Micro Focus liberam você das restrições do hardware e software proprietários do mainframe. Neste guia, o Micro Focus explica como implantar seus aplicativos COBOL e PL/I em execução em mainframes ibm para a nuvem.

### <a name="breathe-new-life-into-mainframes"></a>[Respire vida nova em mainframes](https://www.infosys.com/services/modernization/breathe-new-life-mainframes.html)

 Os mainframes são cada vez mais desafiadores para as empresas manterem. Este white paper da Infosys e da Microsoft destaca a estratégia vencedora para uma migração bem-sucedida do mainframe. O uso de casos e exemplos ilustra as opções.
