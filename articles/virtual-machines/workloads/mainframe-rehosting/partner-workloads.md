# <a name="mainframe-workloads-supported-on-azure"></a>Cargas de trabalho de mainframe suportadas no Azure

As soluções listadas aqui podem ajudá-lo a migrar sua carga de trabalho de mainframe para o Microsoft Azure. Algumas cargas de trabalho podem ser migradas com relativa facilidade. Outras cargas de trabalho que dependem do software do sistema legado podem ser rehospedadas. 

Para obter mais ajuda com a emulação e os serviços do mainframe, consulte o [centro de migração do Mainframe Dozure](https://azure.microsoft.com/migration/mainframe/).

## <a name="migrate-mainframe-closer-to-azure"></a>Migrar mainframe mais perto do Azure

- [Micro Focus Visual COBOL](https://www.microfocus.com/products/visual-cobol/) ferramentas de desenvolvimento e integração.
- [Micro Focus PL/I](https://www.microfocus.com/campaign/download/pli-modernization/) compilador legado para a plataforma .NET, suportando sintaxe PL/I do mainframe, tipos de dados e comportamento.
- Plataforma de integração de mainframe [do Micro Focus Enterprise Server.](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)
- Ferramentas de desenvolvimento e integração [da Fujitsu NetCOBOL.](https://www.fujitsu.com/global/products/software/developer-tool/netcobol/)
- Ferramentas de desenvolvimento e integração [da NTT Data Enterprise COBOL.](https://us.nttdata.com/en/digital/application-development-and-modernization)
- [Compilador legado NTT Open PL/I](https://us.nttdata.com/en/digital/application-development-and-modernization) para a plataforma .NET, suportando sintaxe PL/I do mainframe, tipos de dados e comportamento.
- Ferramentas de desenvolvimento e integração [do compilador COBOL de código de chuva.](https://www.raincode.com/products/cobol/)
- [O compilador PL/I do Raincode](https://www.raincode.com/products/pli/) para a plataforma .NET suporta sintaxe PL/I do mainframe, tipos de dados e comportamento.
- [Compilador ASM370](https://www.raincode.com/technical-landscape/asm370/) de código de chuva para a sintaxe do mainframe Assembler 370 e HLASM.
- [RPG visual ASNA para](https://asna.com/us/products/visual-rpg) compilador .NET para o .NET Framework via plug-in Visual Studio.
- Ferramentas de desenvolvimento e integração da [Modern Systems CTU (COBOL-To-Universal).](https://modernsystems.com/automatic-cobol-to-java-conversion/)

## <a name="deploy-an-emulation-environment-for-online-and-batch-processing"></a>Implantar um ambiente de emulação para processamento on-line e em lote

- Plataforma de integração de mainframe [do Micro Focus Enterprise Server.](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)
- [NTT DATA Mainframe Rehosting Development Environment](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf) ambiente nativo de processamento de transações.
- [NTT DATA Batch Processing Environment](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf) (BPE) incluindo recursos de transação JCL.
- [Emulador CICS de código](https://www.raincode.com/technical-landscape/cics/) de chuva para plataformas .NET e Azure.
- Intérprete JCL compatível com [plugcode JCL.](https://www.raincode.com/products/jcl/)

## <a name="code-conversion"></a>Conversão de código

- Conversão de código fonte [asysco.](https://www.asysco.com/azure-cloud/)
- [O software do LzLabs definiu o](https://www.lzlabs.com/) contêiner de software gerenciado mainframe para migrar aplicativos de mainframe para computadores Linux ou ambientes privados, públicos e híbridos em nuvem.
- [Ferramentas Blu Age](https://www.bluage.com/) para digitalizar aplicativos de negócios e bancos de dados legados.
- [Serviços de Computação](https://www.heirloomcomputing.com/tag/convert-cobol-to-java/) heirloom para converter mainframe COBOL, CICS e VSAM para Java.
