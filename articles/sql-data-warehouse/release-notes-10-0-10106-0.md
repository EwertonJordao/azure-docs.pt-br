# <a name="azure-synapse-analytics-release-notes"></a>Notas de versão do Azure Synapse Analytics

Este artigo resume os novos recursos e melhorias nas versões recentes do [SQL Analytics](sql-data-warehouse-overview-what-is.md) no Azure Synapse. O artigo também lista as atualizações de conteúdo notáveis que não estão diretamente relacionadas à versão, mas publicadas no mesmo período de tempo. Para aprimoramentos em outros serviços do Azure, consulte [atualizações de serviço](https://azure.microsoft.com/updates).

## <a name="check-your-azure-synapse-version"></a>Verifique sua versão Synapse do Azure

À medida que novos recursos são distribuídos para todas as regiões, verifique a versão implantada em sua instância e as notas de versão mais recentes para disponibilidade de recursos. Para verificar a versão, conecte-se ao seu pool do SQL via SQL Server Management Studio (SSMS) e execute `SELECT @@VERSION;` para retornar a versão atual.

Use a versão identificada para confirmar qual versão foi aplicada ao seu pool SQL.

> [!NOTE]
> Em junho de 2020, o nome do produto retornado por SELECT @@VERSION será alterado de SQL Data Warehouse do Microsoft Azure para o Azure Synapse Analytics. Publicaremos a agenda em nossas notas de versão. Essa alteração é relevante para clientes que analisam o nome do produto do resultado de SELECT @@VERSION no código do aplicativo. Para evitar alterações no código do aplicativo devido à Remarcação do produto, use estes comandos para consultar SERVERPROPERTY para o nome e a versão do produto de banco de dados: para retornar o número de versão XX. X. XXXXX. X (sem o nome do produto) Use este comando: 
> ```
> SELECT SERVERPROPERTY('ProductVersion')
> ```
> Para retornar a edição do mecanismo, use este comando que retorna 6 para o Azure Synapse Analytics (anteriormente SQL Data Warehouse): 
> ```
> SELECT SERVERPROPERTY('EngineEdition')
> ```

Para obter mais informações, consulte esta [postagem do blog](https://azure.microsoft.com/blog/smarter-faster-safer-azure-sql-data-warehouse-is-simply-unmatched/)do Azure Synapse.|

O Repositório de Consultas foi projetado para ajudá-lo com a solução de problemas de desempenho de consultas pelo acompanhamento de consultas, planos de consulta, estatísticas de runtime e histórico de consultas para ajudá-lo a monitorar a atividade e o desempenho de seu data warehouse. O Repositório de Consultas é um conjunto de repositórios internos e DMVs (Exibições de Gerenciamento Dinâmico) que permitem:<br/><br/>&bull; &nbsp; identificar e ajustar as principais consultas de consumo de recursos<br/>&bull; &nbsp; identificar e melhorar as cargas de trabalho não planejadas<br/>&bull; &nbsp; avaliar o desempenho e o impacto da consulta no plano por alterações nas estatísticas, nos índices ou no tamanho do sistema (configuração DWU)<br/>&bull; &nbsp; ver o texto completo da consulta para todas as consultas executadas<br/><br/>O Repositório de Consultas contém três repositórios reais:<br/>&bull; &nbsp; um repositório de planos para persistir as informações do plano de execução<br/>&bull; &nbsp; um repositório de estatísticas de tempo de execução para persistir as informações de estatísticas de execução<br/>&bull; &nbsp; um repositório de estatísticas de espera para manter informações de estatísticas de espera.<br/><br/>A análise de SQL no Azure Synapse gerencia esses armazenamentos automaticamente e fornece um número ilimitado de consultas storied nos últimos sete dias sem custo adicional. |**Mesclagem em segundo plano Columnstore**|Por padrão, os dados SQL do Azure armazenam dados no formato colunar, com micro partições chamadas de [RowGroups](sql-data-warehouse-memory-optimizations-for-columnstore-compression.md). Às vezes, devido a restrições de memória no tempo de carregamento de dados ou de build de índice, os rowgroups podem ser compactados com um tamanho menor que o ideal de um milhão de linhas. Os rowgroups também podem se tornar fragmentados devido a exclusões. Rowgroups pequenos ou fragmentados resultam em maior consumo de memória, bem como em uma execução de consulta ineficiente. Com esta versão, a tarefa de manutenção em segundo plano do columnstore mescla pequenos grupos de colunas compactados para criar grupos de rowgroup maiores para utilizar melhor a memória e acelerar a execução da consulta.
