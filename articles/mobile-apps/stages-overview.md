# <a name="different-stages-in-mobile-application-development"></a>Diferentes estágios no desenvolvimento de aplicativos móveis
Construir um aplicativo móvel envolve vários estágios. Você pode facilmente aproveitar os serviços, ferramentas e tecnologias da Microsoft para criar aplicativos móveis de alta classe.

## <a name="app-development"></a>Desenvolvimento de aplicativos
Você pode criar aplicativos clientes nativos de plataforma única com idiomas como Objective-C e Java. Você pode criar aplicativos multiplataforma usando Xamarin, React Native e Unity. Você também pode criar aplicativos híbridos usando cordova. Para construir aplicativos clientes, faça uso de IDEs e editores de código, como Visual Studio, Visual Studio for Mac ou Visual Studio Code em PCs e Macs.

## <a name="plan-and-design-with-back-end-services"></a>Planejar e projetar com serviços back-end
 A Microsoft fornece uma variedade de serviços para configurar um back-end seguro, escalável e confiável para o seu aplicativo móvel. Você pode criar APIs e gatilhos sem servidor sem ter que manter seus próprios servidores. Use notificações push, sincronização de dados offline, autenticação e serviços de dados para formar um back-end móvel abrangente e expansivo que suporte uma variedade de cenários de construção de aplicativos. 
 
 A construção de um serviço back-end tem muitos requisitos. Você precisa:
   - Limitar o armazenamento de dados no dispositivo.
   - Sincronizar dados em vários dispositivos.
   - Certifique-se de que o aplicativo seja executado sem conectividade de rede.
   - Envie notificações aos usuários.
   - Reduza o consumo de bateria.
   - Construa um back-end escalável, confiável e seguro.
   - Automatize a manutenção do servidor.

## <a name="devops-and-continuous-monitoring"></a>DevOps e monitoramento contínuo
Para criar aplicativos móveis e focados em nuvem, aproveite a integração contínua (CI), entrega contínua (CD) e serviços de monitoramento contínuo para produzir builds nativas para todas as plataformas. Você pode executar testes de automação de IU em milhares de dispositivos físicos, automatizar suas versões através de canais de teste beta ou implantar diretamente em lojas de aplicativos. Você também pode coletar informações sobre análise de uso, falha e exceção para seus aplicativos.

## <a name="additional-services"></a>Serviços adicionais
Muitos serviços do Azure podem ser consumidos dependendo do tipo e natureza da aplicação que está sendo construída:
  - Pesquisa Cognitiva do Azure para pesquisar todos os tipos de dados internos, com consulta rápida.
  - Azure Cognitive Services para usar inteligência artificial em sua aplicação.
  - Azure SignalR para permitir a comunicação em tempo real em seu aplicativo.
