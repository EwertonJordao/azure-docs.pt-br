# <a name="yaml-configuration-options-to-customize-the-build-tasks"></a>Opções de configuração YAML para personalizar as tarefas de compilação

Este artigo lista todas as opções de configuração YAML disponíveis em cada uma das tarefas de compilação. O artigo começa com as tarefas para ferramentas de análise de código de segurança. Termina com as tarefas pós-processamento.

## <a name="anti-malware-scanner-task"></a>Tarefa anti-Malware Scanner

| **Inputtype**      | **Tipo**     | **Aplicável**            | **Obrigatório** | **Valor Padrão**             | **Opções (para picklists)**                                   | **Descrição**                                                                                                                                                                                                                                                                                                                            |
|------------|---------------|-----------------------|----------|---------------------------|----------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Inputtype | Picklist | always | True | Basic | Básico, Personalizado | 
| ScanType | Picklist | InputType = Básico | True | PersonalScan | PersonalScan, FullSystemScan, QuickScan, YourConfiguredScan | O tipo de varredura a ser usado para a varredura antimalware.
| FileDirPath | filePath | ScanType = PersonalScan | True | $(Build.StagingDirectory) |  | Indica que o arquivo ou diretório a ser digitalizado.
| Desativação Remediação | booleano | ScanType = PersonalScan | Falso | true |  | Quando verificado: 1) As exclusões de arquivos são ignoradas. 2) Os arquivos de arquivo são digitalizados. 3) As ações não são aplicadas após a detecção. 4) As entradas de registro de eventos não são escritas após a detecção. 5) As detecções da varredura personalizada não são exibidas na interface do usuário. 6) A saída do console mostrará a lista de detecções da varredura personalizada.
| BootSectorScan | booleano | ScanType = PersonalScan | Falso | false |  | Se verificado, ele habilita a varredura do setor de inicialização.
| Argumentos | string | InputType = Personalizado | True | -ScanTypeType 3 -DesativarRemediação -Arquivo $(Build.StagingDirectory) |  | Os argumentos da linha de comando, onde o argumento para -Arquivo é um caminho absoluto ou um caminho relativo para o $(Build.StagingDirectory) predefinido no seu agente de compilação. Você também pode fornecer seus próprios argumentos permitidos pela ferramenta MpCmdRun.exe.<br/><br/>Para obter mais detalhes sobre os argumentos da linha de comando para esta ferramenta, digite <strong>-h</strong> ou <strong>-?</strong> no campo Argumentos e execute a tarefa de compilação.
